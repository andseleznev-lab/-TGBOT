## 🐛 Фаза 3: Bugfix - Кастомная модалка при ошибке (2/2) ✅

- [x] Исправить парсинг JSON + обработку ошибок занятого слота (БЕСПЛАТНЫЕ услуги)
  - **Проблема**: JSON parse error из-за zero-width space → generic alert вместо кастомной модалки
  - **Решение 1**: Очистка невидимых символов перед `JSON.parse()` ✅
    - Регулярка: `/[\u200B-\u200D\uFEFF]/g` (zero-width space, BOM)
    - Строка: `app.js:921-922`
  - **Решение 2**: Упрощение обработки ошибок в `confirmBooking()` ✅
    - Убрана проверка `error.apiResponse?.slot_status`
    - Всегда используется `showSlotTakenPopup(message)` вместо `tg.showAlert`
    - Fallback chain: `error.apiResponse?.message || error.message || дефолт`
    - Строка: `app.js:2127-2133`
  - **Версия**: `app.js?v=50` → `v=51` ✅
  - **Console log**: "version 51 - T-007: Bugfix JSON parse + кастомная модалка" ✅

- [x] Добавить кастомную модалку для ПЛАТНЫХ услуг
  - **Проблема**: При ошибке создания платежа (слот занят) показывается generic `tg.showAlert()` ✅
  - **Решение**: Заменить на `showSlotTakenPopup(message)` в `createPayment()` ✅
    - Используется fallback chain: `error.apiResponse?.message || error.message || дефолт`
    - Удален дублирующий haptic feedback (уже есть в `showSlotTakenPopup()`)
    - Строка: `app.js:373-379`
  - **Версия**: `app.js?v=51` → `v=52` ✅
  - **Console log**: "version 52 - T-007: Кастомная модалка для платных услуг" ✅
  - **Commit**: `a9ddee2` - [T-007] FIX: Кастомная модалка для платных услуг ✅
