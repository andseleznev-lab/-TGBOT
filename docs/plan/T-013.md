## Архитектура решения

### Структура данных

```
club_payments          club_meetings              club_registrations
──────────────         ──────────────             ──────────────────────
id                     id  ◄──────────────────    id
payment_id             meeting_date (UNIQUE)   │  meeting_id → club_meetings.id
telegram_user_id ──►   zoom_meeting_id         │  telegram_user_id
amount                 created_at              │  zoom_join_url  (уникальная!)
status                                         └─ created_at
zoom_link  (legacy)                               UNIQUE(meeting_id, telegram_user_id)
created_at
```

**Поток Make.com (еженедельно):**
```
Schedule trigger
  → Zoom API: Create Meeting
  → INSERT club_meetings
  → SELECT club_payments WHERE status='succeeded'
  → FOR EACH user: Zoom API Add Registrant → unique join_url
  → INSERT club_registrations (ON CONFLICT DO NOTHING)
```

**Поток Make.com (новая оплата):**
```
Payment succeeded
  → INSERT club_payments
  → SELECT next meeting FROM club_meetings WHERE date >= today
  → IF meeting exists: Zoom Add Registrant → INSERT club_registrations
```

**Поток бота:**
```
loadClubData()
  → Supabase: SELECT club_payments WHERE telegram_user_id = USER.id     [уже есть]
  → Supabase: SELECT club_meetings WHERE meeting_date >= -30 days        [новое]
  → Supabase: SELECT club_registrations WHERE telegram_user_id = USER.id [новое]
  → State.clubMeetings = meetings
  → State.clubRegistrations = registrations (map: meeting_id → zoom_join_url)

renderClubScreen()
  → Итерируем State.clubMeetings (вместо расчётных воскресений)
  → Для каждой встречи: ищем zoom_join_url в State.clubRegistrations
  → renderClubMeetingCard({ date, zoomLink: url || null })
```

## Компоненты и функции

### 1. Новая SQL-миграция

**Файл**: `docs/supabase-migration-club-meetings.sql`

```sql
-- Таблица встреч клуба
CREATE TABLE IF NOT EXISTS club_meetings (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  meeting_date  DATE NOT NULL,
  zoom_meeting_id TEXT NOT NULL,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT unique_meeting_date UNIQUE (meeting_date)
);

-- Таблица регистраций участников
CREATE TABLE IF NOT EXISTS club_registrations (
  id               UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  meeting_id       UUID NOT NULL REFERENCES club_meetings(id) ON DELETE CASCADE,
  telegram_user_id BIGINT NOT NULL,
  zoom_join_url    TEXT NOT NULL,
  created_at       TIMESTAMPTZ DEFAULT NOW(),
  CONSTRAINT unique_member_meeting UNIQUE (meeting_id, telegram_user_id)
);

-- Индексы
CREATE INDEX IF NOT EXISTS idx_club_meetings_date
  ON club_meetings(meeting_date DESC);

CREATE INDEX IF NOT EXISTS idx_club_registrations_user
  ON club_registrations(telegram_user_id);

CREATE INDEX IF NOT EXISTS idx_club_registrations_meeting
  ON club_registrations(meeting_id);

-- RLS
ALTER TABLE club_meetings ENABLE ROW LEVEL SECURITY;
ALTER TABLE club_registrations ENABLE ROW LEVEL SECURITY;

-- club_meetings: публичное чтение (anon видит расписание)
CREATE POLICY "Allow public read club_meetings"
  ON club_meetings FOR SELECT USING (true);

CREATE POLICY "Deny public insert club_meetings"
  ON club_meetings FOR INSERT WITH CHECK (false);

-- club_registrations: публичное чтение (фильтрация по telegram_user_id на клиенте)
CREATE POLICY "Allow public read club_registrations"
  ON club_registrations FOR SELECT USING (true);

CREATE POLICY "Deny public insert club_registrations"
  ON club_registrations FOR INSERT WITH CHECK (false);
```

### 3. app.js — loadClubData() (~строка 3440)

**Добавить после загрузки payments** (в Supabase-ветке):

```javascript
// Загружаем расписание встреч (ближайшие + последние 4 недели)
const cutoffDate = new Date();
cutoffDate.setDate(cutoffDate.getDate() - 28);
const cutoffStr = cutoffDate.toISOString().slice(0, 10);

const { data: meetings, error: meetingsError } = await supabaseClient
    .from('club_meetings')
    .select('id, meeting_date')
    .gte('meeting_date', cutoffStr)
    .order('meeting_date');

if (!meetingsError && meetings) {
    State.clubMeetings = meetings;
    console.log(`📅 [loadClubData] Встречи: ${meetings.length} записей`);
}

// Загружаем регистрации текущего пользователя
const { data: registrations, error: regError } = await supabaseClient
    .from('club_registrations')
    .select('meeting_id, zoom_join_url')
    .eq('telegram_user_id', USER.id);

if (!regError && registrations) {
    // Map: meeting_id → zoom_join_url для быстрого lookup
    State.clubRegistrations = Object.fromEntries(
        registrations.map(r => [r.meeting_id, r.zoom_join_url])
    );
    console.log(`🔗 [loadClubData] Регистрации: ${registrations.length} ссылок`);
}
```

```javascript
    if (daysUntilSunday > 0) { ... }

// Станет:
function getNextWednesdays(startDate, count = 4) {
    // ...
    // Находим ближайшую среду (день недели 3)
    const daysUntilWednesday = (3 - current.getDay() + 7) % 7;
    if (daysUntilWednesday > 0) {
        current.setDate(current.getDate() + daysUntilWednesday);
    } else if (current.getDay() === 3) {
        current.setDate(current.getDate() + 7); // уже среда — берём следующую
    }
    // Собираем N сред
    for (let i = 0; i < count; i++) {
        wednesdays.push(new Date(current));
        current.setDate(current.getDate() + 7);
    }
    return wednesdays;
```

### 8. Make.com — Новый сценарий «Клуб: еженедельный Zoom»

**(Ручная настройка в Make.com)**

**Расчёт даты и времени встречи:**

Сценарий запускается каждый **понедельник** → `now` = понедельник → `addDays(now; 2)` = ближайшая среда.

Формула для поля **Start Date and Time** в модуле Zoom (аналогично существующей формуле для консультаций, но без Google Календаря как источника):

```
{{formatDate(addDays(now; 2); "YYYY-MM-DD")}}T19:00:00+03:00
```

- `formatDate(addDays(now; 2); "YYYY-MM-DD")` — дата ближайшей среды
- `T19:00:00+03:00` — время 19:00 с явным МСК-смещением

Формат `+03:00` аналогичен существующей формуле консультаций (`"YYYY-MM-DDTHH:mm:ss+03:00"`), чтобы Zoom корректно определил время. Timezone в модуле также ставим **Moscow**.

Для поля `meeting_date` в Supabase нужна только дата:
```
{{formatDate(addDays(now; 2); "YYYY-MM-DD")}}
```

**Шаги сценария:**

1. **Trigger**: Schedule — каждый понедельник, 10:00 МСК
2. **Zoom: Create Meeting**
   - Topic: `Клуб — {{formatDate(addDays(now; 2); "DD.MM")}}`  (например: «Клуб — 11.03»)
   - Type: Scheduled Meeting
   - Start Date and Time: `{{formatDate(addDays(now; 2); "YYYY-MM-DD")}}T19:00:00+03:00`
   - Duration: 90 мин
   - Timezone: Moscow
   - Approval Type: **Automatically Approve** ← включает регистрацию, уникальные ссылки
   - Registrants Email Notification: **No** ← не спамить участников
   - → Получаем `id` (zoom_meeting_id)
3. **Supabase HTTP: INSERT club_meetings**
   - `POST /rest/v1/club_meetings`
   - Body: `{ "meeting_date": "{{formatDate(addDays(now; 2); "YYYY-MM-DD")}}", "zoom_meeting_id": "{{id}}" }`
   - Header: `Prefer: resolution=merge-duplicates` (ON CONFLICT UPDATE — безопасный ретрай)
   - → Получаем `id` новой записи (meeting_uuid)
4. **Supabase HTTP: SELECT участников**
   - `GET /rest/v1/club_payments?status=eq.succeeded&select=telegram_user_id`
   - → Массив `telegram_user_id`
5. **Iterator**: по каждому участнику
6. **Zoom: Add Meeting Registrant**
   - Meeting ID: `{{zoom_meeting_id из шага 2}}`
   - Email: `{{telegram_user_id}}@club.local` (Zoom требует email, письма не отправляются т.к. шаг 2)
   - First Name: `Участник`
   - → `join_url` (уникальный для этого участника)
7. **Supabase HTTP: INSERT club_registrations**
   - `POST /rest/v1/club_registrations`
   - Body: `{ "meeting_id": "{{meeting_uuid}}", "telegram_user_id": {{telegram_user_id}}, "zoom_join_url": "{{join_url}}" }`
   - Header: `Prefer: resolution=merge-duplicates`
8. **Telegram: Send a Message** (в том же iterator-цикле, после шага 7)
   - Chat ID: `{{telegram_user_id}}`
   - Текст: ссылка на встречу + дата
   - Пример: `«Встреча клуба в среду {{formatDate(addDays(now; 2); "DD.MM")}} в 19:00 — ваша личная ссылка: {{join_url}}»`
   - Настраивается отдельно (пользователь делает сам)

### 9. Make.com — Правка payment-сценария

После INSERT нового succeeded-платежа добавить:

1. **Supabase HTTP: SELECT ближайшая встреча**

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Zoom требует реальный email для регистрации | Средняя | Высокое | Использовать `{telegram_id}@tgclub.local` или хранить email из club_payments |
| Make.com iterator медленный при >100 участниках | Низкая | Низкое | Разбить на batches или использовать Zoom bulk registration |
| Дубли при ретрае Make.com | Средняя | Низкое | ON CONFLICT DO NOTHING / merge-duplicates |
| Supabase anon key читает чужие zoom_join_url | Средняя | Низкое | join_url не секрет — просто ссылка в конкретный Zoom |
