## Краткое описание

Добавить поле ввода email на экран бронирования (только для платных услуг). Email валидируется, персистируется в localStorage, передаётся в `createPayment()` → Make.com → YooKassa как `receipt.customer.email`.

## Архитектура решения

### Обзор

1. `State.userEmail` хранит текущий email сессии
2. `renderBookingScreen()` рендерит поле email над кнопкой "Подтвердить" (только если `isPaidService`)
3. `confirmBooking()` валидирует email перед созданием платежа
4. `createPayment(slotId, serviceId, email)` принимает email и передаёт в API
5. Make.com добавляет `receipt` в HTTP body к YooKassa на основе `metadata.user_email`

## Компоненты и функции

### Frontend компоненты

#### Изменение: `State`

```javascript
const State = {
  // ... существующие поля ...
  userEmail: null,   // Email для фискального чека (берётся из localStorage при старте)
};
```

При инициализации приложения восстанавливать из localStorage:
```javascript
State.userEmail = localStorage.getItem('user_email') || null;
```

---

#### Изменение: `renderBookingScreen()`

Добавить HTML-блок email-инпута **между** секцией слотов и кнопкой "Подтвердить":

```javascript
// Определяем платность услуги (price > 0)
const price = CONFIG.SERVICE_PRICES[State.selectedService];
const isPaidService = price && price > 0;

// Блок email (только для платных услуг и когда слот выбран)
${State.selectedSlot && isPaidService ? `
    <div class="email-input-section">
        <label class="email-label">Email для чека</label>
        <input
            type="email"
            class="email-input"
            placeholder="example@mail.ru"
            value="${escapeHtml(State.userEmail || '')}"
            oninput="State.userEmail = this.value.trim()"
            autocomplete="email"
        />
    </div>
` : ''}

${State.selectedSlot ? `
    <button class="confirm-button glass-card" onclick="confirmBooking()">
        Подтвердить запись
    </button>
` : ''}
```

---

#### Изменение: `confirmBooking()`

Для платных услуг добавить валидацию email **перед** вызовом `createPayment()`:

```javascript
if (isPaidService) {
    // Валидация email
    const email = (State.userEmail || '').trim();
    const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
    if (!email) {
        tg.HapticFeedback.notificationOccurred('error');
        showSlotTakenPopup('Введите email для получения чека');
        return;
    }
    if (!emailRegex.test(email)) {
        tg.HapticFeedback.notificationOccurred('error');
        showSlotTakenPopup('Введите корректный email');
        return;
    }
    // Сохраняем в localStorage
    localStorage.setItem('user_email', email);

    // ... далее существующий код createPayment(slot.id, State.selectedService, email)
}
```

---

#### Изменение: `createPayment(slotId, serviceId, email)`

Добавить параметр `email`, включить в payload:

```javascript
async function createPayment(slotId, serviceId, email) {
    // ...
    const result = await BookingAPI.request('create_payment', {
        slot_id: slotId,
        service: serviceId,
        user_email: email   // ← новое поле
    });
    // ...
}
```

### Backend компоненты (Make.com)

В HTTP module запроса к YooKassa добавить в Body блок `receipt`:

```json
{
  "amount": { "value": "...", "currency": "RUB" },
  "capture": true,
  "confirmation": { "type": "redirect", "return_url": "..." },
  "description": "Консультация психолога",
  "metadata": { ... },
  "receipt": {
    "customer": {
      "email": "{{metadata.user_email}}"
    },
    "items": [
      {
        "description": "{{description}}",
        "quantity": "1.00",
        "amount": {
          "value": "{{amount.value}}",
          "currency": "RUB"
        },
        "vat_code": 1
      }
    ]
  }
}
```

> ⚠️ `vat_code: 1` — без НДС. Уточнить у владельца магазина правильный код.

## Потоки данных

### Основной поток (Happy Path)

1. Пользователь выбирает дату и время платной услуги
   ↓
2. `renderBookingScreen()` — показывает email-инпут (pre-fill из localStorage)
   ↓
3. Пользователь вводит email, нажимает "Подтвердить запись"
   ↓
4. `confirmBooking()`:
   - проверяет `State.userEmail` через regex
   - сохраняет в localStorage
   - вызывает `createPayment(slot.id, service, email)`
   ↓
5. `createPayment()` → POST к Make.com с `{ slot_id, service, user_email }`
   ↓
6. Make.com → POST к YooKassa с `receipt.customer.email`
   ↓
7. YooKassa возвращает `payment_url`
   ↓
8. Frontend показывает `showPaymentConfirmModal()`

## Безопасность

**Frontend**:
- Email выводится в HTML только через `escapeHtml()`
- Email не логируется явно (допустимо логировать `email.replace(/@.*/, '@***')`)

**Backend (Make.com)**:
- Email используется только в `receipt.customer.email`, не в URL и не в description

- [ ] После перезагрузки email подставляется автоматически
- [ ] YooKassa принимает платёж без ошибки `receipt`
- [ ] Чек отправляется на указанный email

## Риски и митигация

**Риск 1: vat_code не совпадает с настройками магазина**
- Вероятность: Средняя
- Митигация: Проверить в ЛК YooKassa раздел «Настройки → Чеки»

- Вероятность: Низкая
- Митигация: Проверить в Make.com логах тело запроса к YooKassa
