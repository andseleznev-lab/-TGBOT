## Краткое описание

Два небольших UI-улучшения: добавить визуальную угловую рамку (класс `screen-title--diagnostic`) к заголовкам вкладок «Мои записи» и «Клуб», а также реализовать автоматическое переключение календаря на первый доступный месяц, если в текущем месяце нет слотов.

---

## Проблема / Возможность

### Контекст

После тикета T-009 угловая рамка появилась только на экране «Диагностика». Остальные вкладки выглядят менее презентабельно. Кроме того, в конце месяца пользователи видят пустой календарь и должны вручную нажимать «›» для перехода на следующий месяц.

### Текущая ситуация (AS IS)

- `.screen-title--diagnostic` с угловыми псевдоэлементами `::before`/`::after` применён только на экране «Диагностика»
- Вкладки «Мои записи» и «Клуб» используют обычный `screen-title` без рамки
- При выборе услуги в конце месяца `State.currentMonth` остаётся текущим месяцем, даже если все слоты — в следующем; пользователь видит пустой календарь

### Желаемая ситуация (TO BE)

- Угловая рамка отображается на заголовках «Мои записи» и «Клуб» (все состояния) — визуальная консистентность
- После загрузки дат система автоматически переключает календарь на первый доступный месяц, если в текущем месяце слотов нет

## Требования

### Функциональные требования

#### Must Have

- [ ] Добавить класс `screen-title--diagnostic` к заголовку «Мои записи» (все состояния: загрузка, есть записи, нет записей)
- [ ] Добавить класс `screen-title--diagnostic` к заголовку «Клуб» (все состояния: загрузка, есть подписка, нет подписки, заголовок встречи)
- [ ] В `onServiceSelect` после `loadAvailableDates`: если в текущем месяце нет слотов — установить `State.currentMonth` на месяц первой доступной даты

#### Should Have

- [ ] Не изменять CSS — класс уже существует, повторное использование без дублирования стилей

### Не функциональные требования (NFR)

**Производительность**:

- Проверка текущего месяца — O(n) по availableDates, не добавляет API-вызовов

**Безопасность**:

- Нет новых входных данных, нет рисков

**UX/UI**:

- Рамка должна выглядеть идентично экрану «Диагностика»
- Автопереключение должно быть незаметным — пользователь сразу видит нужный месяц

## Сценарии использования

### Основной сценарий (Happy Path) — Автопереключение

**Действующие лица**: Пользователь, Frontend

**Предусловия**:

- Текущий месяц — февраль, все доступные слоты — в марте

**Шаги**:

1. Пользователь выбирает услугу
2. Система вызывает `onServiceSelect`, очищает состояние, рендерит экран
3. Система загружает `availableDates` через `loadAvailableDates`
4. После загрузки: проверяется наличие слотов в феврале → слотов нет
5. Система находит первую дату в `availableDates[0]` → март
6. `State.currentMonth` устанавливается в март
7. Вызывается `renderBookingScreen()` — календарь сразу открыт на марте

**Результат**:

- Пользователь видит календарь с доступными датами без ручного переключения

### Основной сценарий (Happy Path) — Угловая рамка

**Шаги**:

1. Пользователь открывает вкладку «Мои записи» или «Клуб»
2. Заголовок рендерится с классами `screen-title screen-title--diagnostic fade-in`
3. CSS псевдоэлементы `::before`/`::after` добавляют угловую рамку

**Результат**:

- Вкладки выглядят визуально единообразно с «Диагностикой»

### Исключительные ситуации (Edge Cases)

**Ошибка 1**: `availableDates` пустой после загрузки
- **Причина**: нет доступных слотов
- **Обработка**: блок автопереключения не выполняется (`if (State.availableDates.length > 0)`)
- **Сообщение пользователю**: пустой календарь текущего месяца (без ошибки)

**Ошибка 2**: Все слоты в текущем месяце присутствуют
- **Причина**: штатная ситуация
- **Обработка**: `hasCurrentMonthSlots === true` → `State.currentMonth` не меняется

**Ошибка 3**: Все слоты в прошлых месяцах
- **Причина**: устаревшие данные
- **Обработка**: `availableDates[0]` будет датой в прошлом → переключится на этот месяц; отображение пустого календаря

## UI/UX Дизайн

### Основные элементы интерфейса

- **Экран «Мои записи»**: `<h1 class="screen-title screen-title--diagnostic fade-in">Мои записи</h1>`
- **Экран «Клуб»** (все состояния): `<h1 class="screen-title screen-title--diagnostic fade-in">Встречи клуба...</h1>`
- **Угловая рамка**: реализована через CSS `::before`/`::after` — цвет `#D4A574`, размер `50px × 50px`

### Взаимодействия

- **Угловая рамка**: статическая, без интерактивности
- **Автопереключение**: происходит незаметно, до первого рендера экрана бронирования

## Риски и ограничения

### Технические риски

- **Риск 1**: Формат даты `DD.MM.YYYY` в `availableDates` — парсинг хрупкий
  - **Вероятность**: Низкая
  - **Влияние**: Среднее
  - **Митигация**: Строгий split('.') с индексами `[dd, mm, yyyy]`, покрыт тестом на реальных данных

### Ограничения

- Логика автопереключения опирается на сортировку `availableDates` — первый элемент массива должен быть наименьшей датой (выполняется по контракту `loadAvailableDates`)
